## ğŸ“ Lessons Learned

### Technical Insights

**1. Python Distribution Matters**
- msys64 Python: Good for system utilities, bad for scientific computing
- Official Python: Best for development, full wheel support
- Embeddable Python: Good for distribution, bad for development (no tkinter)

**2. Package Installation Strategy**
- Pre-built wheels >> Compilation from source
- SSL certificates crucial for pip
- Standard Python has best PyPI compatibility

**3. PyInstaller with Heavy Dependencies**
- Hidden imports critical for matplotlib, sklearn
- `collect_data_files()` needed for package data
- Build time scales with package size (~2 min for v0.6)

**4. Graceful Degradation Success**
- Feature flags allow development without all dependencies
- Try/except imports prevent crashes
- User-friendly warnings guide users

### Development Process

**What Worked Well:**
- Architectural planning before implementation
- Complete backend before UI integration
- Comprehensive documentation
- Feature detection and fallbacks

**What Could Be Improved:**
- Environment validation earlier in development
- Package dependency testing before coding
- CI/CD pipeline for automated builds
- Automated testing of EXE
